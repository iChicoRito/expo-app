# Objective: SVG Characters for Onboarding Screens

---

## Description

This objective adds SVG character illustrations to each of the three onboarding screens in the Spillr app. Each page has a designated SVG asset that must be placed above the existing text content and horizontally centered on the screen. No new pages or layout restructuring is required — the characters are additive to the current onboarding screen structure.

---

## Objectives Breakdown

### 1. Primary Objective

Place the correct SVG character asset on each of the three onboarding screens, positioned above the text content and horizontally centered.

---

### 2. Secondary Objectives

_Not applicable based on provided input._

---

### 3. Supporting Tasks

#### 3.1 SVG Asset Placement per Page

- On the **1st onboarding page**, use the SVG at `assets/svg/on-boarding/1st.svg`.
- On the **2nd onboarding page**, use the SVG at `assets/svg/on-boarding/2nd.svg`.
- On the **3rd onboarding page**, use the SVG at `assets/svg/on-boarding/3rd.svg`.

#### 3.2 Positioning Requirements

- Each SVG character must be placed **above** the text content on its respective page.
- Each SVG character must be **horizontally centered** on the screen.
- These positioning rules apply uniformly to all three pages.

---

### 4. Detailed Breakdown

#### 4.1 Asset-to-Page Mapping

Each onboarding page has a strictly assigned SVG. Assets must not be swapped or reused across pages:

| Page                | Asset Path                       |
| ------------------- | -------------------------------- |
| 1st Onboarding Page | `assets/svg/on-boarding/1st.svg` |
| 2nd Onboarding Page | `assets/svg/on-boarding/2nd.svg` |
| 3rd Onboarding Page | `assets/svg/on-boarding/3rd.svg` |

#### 4.2 Layout Position

The SVG characters must render above the text content block on each screen and be horizontally centered relative to the screen width. No specific sizing, padding, or spacing values were provided in the input — these are implementation details to be determined during development while respecting the existing onboarding layout.
