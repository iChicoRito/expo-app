# Sound Effects for Specific Pages Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Play three `.mp3` sound effects on the Preparation, Streak, and Results pages, each tied to a specific animation or visibility event, with a one-time guard where looping animations are involved.

**Architecture:** Extend the existing `SfxKey` union and `SFX_SOURCES` map in `contexts/audio-store.tsx` with two new keys (`"preparation-star"` and `"spill-result"`). Then call `playSfx` in each target page using `useEffect` (preparation, results) or `useFocusEffect` (streak) to match the timing constraint per page. A `useRef` boolean guard in `results.tsx` enforces one-time playback independent of Lottie loop iterations.

**Tech Stack:** Expo SDK 54, React 19, expo-av (Audio.Sound), lottie-react-native, expo-router (useFocusEffect), React hooks (useEffect, useRef, useCallback)

### Task 4: Results page — spill-result SFX with one-time guard

**Files:**
- Modify: `app/results.tsx`

The result Lottie may loop (`loop={answeredCount === 0}` for the ghosted scenario). The SFX must only play once per page visit. The one-time guard is a `useRef<boolean>` that is set on first play and never reset — this is independent of the `useEffect` dependency array, ensuring robustness even if `playSfx` were to change reference.

- [ ] **Step 1: Add useRef import and one-time guard to results.tsx**

The `useRef` import is already available via `"react"` if not already imported. Check the current import:
```typescript
import { useCallback, useEffect } from "react";
```
Update to include `useRef`:
```typescript
import { useCallback, useEffect, useRef } from "react";
```

- [ ] **Step 2: Add the spill-result SFX effect inside ResultsScreen**

Inside `ResultsScreen`, after the existing `const { playSfx, stopIngameBgm } = useAudioStore();` line, add the ref and effect:
```typescript
const spillSoundPlayedRef = useRef(false);

useEffect(() => {
  if (!spillSoundPlayedRef.current) {
    spillSoundPlayedRef.current = true;
    playSfx("spill-result");
  }
}, [playSfx]);
```

This ensures `spill-result` plays exactly once per component mount, regardless of how many times the Lottie loops.
